# Open Brain — Web Control Panel PRD

## Overview

Эволюция веб-интерфейса Open Brain из окна наблюдателя в полноценную панель управления вторым мозгом. Текущий UI позволяет только смотреть (поиск, список, статистика). Цель — дать возможность редактировать, удалять, рефлексировать и понимать структуру своего мышления.

**Статус:** Draft
**Дата:** 2026-03-04
**Связанный документ:** [PRD.md](./PRD.md)

---

## Принципы

1. **Сад, не склад** — знания требуют ухода, прополки и полива
2. **Знание имеет структуру** — не все мысли одинаково важны, истинны или актуальны
3. **Рефлексия как фича** — второй мозг должен помогать думать, а не только помнить
4. **Прямое манипулирование** — любое действие доступно в 1-2 клика прямо в контексте

---

## Идеи

Шкала сложности:

| Уровень | Описание | Ориентир |
|---------|----------|----------|
| **S** | Тривиально | 1-2 часа. Один API-эндпоинт + немного JS |
| **M** | Умеренно | 3-6 часов. Новый UI-компонент + несколько эндпоинтов |
| **L** | Существенно | 1-2 дня. Новая вкладка/экран, изменения в схеме БД |
| **XL** | Масштабно | 2-5 дней. AI-пайплайн, визуализация, фоновые процессы |

### Раздел 1: Философ — «Забвение, вес и ритуал»

#### 1.1 Fade / Amplify (управление весом мысли)

**Суть:** Кнопки на каждой карточке мысли: «fade» (понизить значимость) и «amplify» (повысить). Faded-мысли опускаются ниже в поиске, amplified — поднимаются. Как в настоящей памяти: важное усиливается, неважное тускнеет.

**Реализация:**
- Новое поле `weight` (float, default 1.0) в таблице thoughts
- Миграция БД: `ALTER TABLE ADD COLUMN weight`
- API: `PATCH /api/thoughts/:id/weight` с body `{ direction: "fade" | "amplify" }`
- Fade: `weight *= 0.5`, Amplify: `weight *= 2.0` (с границами 0.1 — 10.0)
- Поисковый запрос: `similarity * weight` вместо голого `similarity`
- UI: две иконки на карточке (стрелка вниз / стрелка вверх), визуальная opacity привязана к weight

**Сложность: M**
Миграция простая, но нужно менять поисковый запрос в repository и добавить визуальную индикацию в UI.

---

#### 1.2 Компостная яма (Compost)

**Суть:** Отдельное пространство для мыслей, которые ты сознательно отпускаешь. Не корзина (мгновенное удаление) — а ритуал прощания. Через 30 дней мысли из компоста растворяются навсегда.

**Реализация:**
- Новое поле `composted_at` (timestamp, nullable) в таблице thoughts
- API: `POST /api/thoughts/:id/compost` — отправить в компост
- API: `POST /api/thoughts/:id/restore` — вернуть из компоста
- API: `GET /api/compost` — список компостируемых мыслей с обратным отсчётом
- Cron-job или cleanup при старте: удалить записи где `composted_at < now() - 30 days`
- Composted мысли исключаются из поиска и recent
- UI: новая вкладка «Compost» с визуальным таймером дней до растворения
- UI: кнопка «compost» на карточке мысли (иконка листа/растения)

**Сложность: M**
Основная работа — новая вкладка и cleanup-логика. Схема и API простые.

---

#### 1.3 Еженедельная рефлексия (Weekly Review)

**Суть:** Экран «Вот что ты думал неделю назад. Это всё ещё актуально?» Не push-уведомление, а приглашение к диалогу с прошлым собой. Показывает случайную выборку мыслей 7-дневной давности и предлагает действия: подтвердить, развить, отпустить.

**Реализация:**
- API: `GET /api/reflection/weekly` — возвращает 5-7 мыслей из диапазона 6-8 дней назад
- UI: отдельный экран/вкладка с карточками в формате «Неделю назад ты думал:»
- На каждой карточке три действия:
  - «Still true» → amplify
  - «Evolved» → открывает inline-редактор для обновления
  - «Let go» → compost
- Индикатор на главной: бейдж «Weekly review» если есть непросмотренные

**Сложность: M**
API тривиальный (select по дате), основная работа — новый UI-экран с тремя действиями.

### Раздел 2: Эксперт по PKM — «Хирургия и граф»

#### 2.1 Удаление мысли из UI

**Суть:** Базовая операция. Кнопка «Delete» на каждой карточке с подтверждением.

**Реализация:**
- API: `DELETE /api/thoughts/:id` (уже есть в repository, только эндпоинт нужен)
- UI: иконка корзины на карточке → confirm-диалог → удаление → карточка исчезает
- Анимация исчезновения (opacity transition)

**Сложность: S**
Repository метод уже есть. Нужен один API-эндпоинт + JS в UI.

---

#### 2.2 Inline-редактирование

**Суть:** Клик на мысль → контент становится редактируемым textarea. Сохранение → пересчёт эмбеддинга и метаданных.

**Реализация:**
- API: `PUT /api/thoughts/:id` с body `{ content, title, tags }`
- Backend: обновление записи + `embedding = await embed(newContent)` + re-extract metadata
- Repository: новый метод `update(id, fields)` (или расширить существующий)
- UI: клик на контент → textarea, клик на title → input, теги → editable chips
- UI: кнопки «Save» / «Cancel», индикатор «re-embedding...»

**Сложность: M**
Основная сложность — пересчёт эмбеддинга при изменении контента и хороший inline-editing UX.

---

#### 2.3 Batch-операции (массовые действия)

**Суть:** Чекбоксы на карточках, toolbar сверху: «Выбрано N → удалить / добавить тег / compost / сменить source».

**Реализация:**
- UI: чекбокс на каждой карточке, floating toolbar при selection > 0
- API: `POST /api/thoughts/batch` с body `{ ids: [...], action: "delete" | "tag" | "compost" | "retag", params: {} }`
- Backend: транзакция для batch-операций
- Действия: delete, add_tag, remove_tag, compost, change_source
- UI: «Select all on page» / «Deselect all»

**Сложность: L**
Много UI-работы (selection state, toolbar, разные модалки для разных действий) + batch API с транзакциями.

---

#### 2.4 Карта связей (Knowledge Graph)

**Суть:** Визуализация: каждая мысль — точка на 2D-плоскости. Позиция определяется эмбеддингами (t-SNE / UMAP). Близкие мысли рядом. Кластеры подсвечены. Клик — детали мысли. Zoom — от overview до отдельных мыслей.

**Реализация:**
- Backend: эндпоинт `GET /api/graph` — возвращает все мысли с координатами
- Dimension reduction: t-SNE или UMAP для проекции 1536-dim эмбеддингов в 2D
  - Вариант A: считать на бэкенде (библиотека druid.js или вызов Python-скрипта)
  - Вариант B: предрассчитывать при индексации и хранить x,y в БД
- Frontend: Canvas/WebGL рендеринг (d3-force или pixi.js)
- Кластеризация: DBSCAN или k-means по эмбеддингам, цветовая кодировка
- Интерактив: zoom, pan, клик на точку → sidebar с деталями, hover → title

**Сложность: XL**
Dimension reduction — нетривиальная задача. Визуализация сотен/тысяч точек требует Canvas/WebGL. Много UX-работы (zoom, pan, tooltips, clusters).

---

#### 2.5 Детектор дубликатов

**Суть:** Экран, показывающий пары мыслей с cosine similarity > 0.92. Для каждой пары: «объединить», «удалить одну», «оставить обе».

**Реализация:**
- API: `GET /api/duplicates?threshold=0.92&limit=20`
- Backend: SQL-запрос с cross-join по cosine similarity (тяжёлый на больших объёмах)
  - Оптимизация: считать только для мыслей из одного source или за последние 90 дней
- UI: карточки-пары, similarity score, три кнопки действий
- «Merge» — объединяет контент двух мыслей в одну, пересчитывает эмбеддинг

**Сложность: L**
SQL-запрос для поиска похожих пар требует оптимизации. Merge-логика нетривиальна (какой контент оставить, как объединить теги).

---

#### 2.6 Orphan-теги

**Суть:** Экран «Теги, используемые только один раз». Это или уникальные инсайты, или мусор — человек должен решить.

**Реализация:**
- API: `GET /api/tags/orphans` — теги с count = 1
- UI: список orphan-тегов, для каждого — мысль, которая его использует
- Действия: «удалить тег», «переименовать», «оставить»
- Переиспользует существующий `repository.listTags()` с фильтром

**Сложность: S**
Данные уже есть (listTags возвращает count). Нужен фильтр + простой UI.

### Раздел 3: Гносеолог — «Структура знания»

#### 3.1 Эпистемический статус

**Суть:** Каждая мысль получает метку epistemic status:
- `hypothesis` — думаю так, но не уверен
- `conviction` — верю в это
- `fact` — проверено
- `outdated` — было верно, уже нет
- `question` — не знаю и хочу узнать

**Реализация:**
- Новое поле `epistemic_status` (enum, default null) в таблице thoughts
- Миграция БД
- API: `PATCH /api/thoughts/:id/status` с body `{ status: "hypothesis" | ... }`
- UI: dropdown или набор иконок на каждой карточке
- Фильтры по статусу на вкладках Search и Recent
- Специальные экраны:
  - «Все твои нерешённые вопросы» (`status = 'question'`)
  - «Убеждения старше 6 месяцев» (`status = 'conviction' AND created_at < now() - 6m`)

**Сложность: M**
Поле и API простые. Основная работа — хороший UX для выбора статуса и фильтры.

---

#### 3.2 Линия времени (Belief Timeline)

**Суть:** Хронологическая визуализация: как менялись мысли по одной теме. Поиск «продуктивность» → таймлайн: январь — «режим дня важен», март — «энергия важнее», июнь — «сон + движение = энергия». Показывает эволюцию мышления, не снимок.

**Реализация:**
- API: `GET /api/timeline?q=<topic>&limit=30` — семантический поиск, результаты отсортированы по дате (не по similarity)
- UI: вертикальный таймлайн с карточками слева/справа
- Визуализация: линия времени с точками, при наведении — контент
- Фильтры: период, минимальная similarity
- Опционально: AI-summary эволюции («Твоё мнение о X сместилось от A к B»)

**Сложность: L**
Сам запрос — вариация существующего search с другой сортировкой. Основная работа — timeline UI-компонент и UX перемотки времени.

---

#### 3.3 Детектор противоречий (Contradiction Finder)

**Суть:** Автоматический поиск: «ты сказал X здесь и не-X там». Экран противоречий как приглашение к более глубокому мышлению. Твой внутренний Сократ.

**Реализация:**
- Backend: для каждой мысли найти семантически близкие, затем через LLM определить, противоречат ли они
- API: `GET /api/contradictions?limit=10`
- Pipeline:
  1. Выбрать пары с high similarity (>0.7) но разных дат
  2. Отправить пары в GPT-4o-mini: «Эти две мысли противоречат друг другу? Ответь yes/no и почему»
  3. Кэшировать результаты (таблица `contradiction_checks`)
- UI: карточки-пары с объяснением противоречия
- Действия: «разрешить» (написать новую мысль-синтез), «не противоречие» (dismiss), «оставить»
- Фоновый процесс: проверять новые мысли против существующих при сохранении

**Сложность: XL**
LLM-пайплайн для анализа пар, кэширование результатов, фоновая обработка, нетривиальный UX «разрешения» противоречий.

---

#### 3.4 Сила убеждения (Conviction Strength)

**Суть:** Визуализация: «О чём ты думаешь чаще, чем тебе кажется?» Автоматическая кластеризация мыслей по темам + подсчёт частоты. Одна мысль о квантовой физике — любопытство. Двадцать — одержимость.

**Реализация:**
- Backend: кластеризация эмбеддингов (k-means или DBSCAN)
- Автоматическое именование кластеров через LLM: «О чём этот кластер из N мыслей?»
- API: `GET /api/clusters` — список кластеров с названиями, размерами и примерами
- UI: bubble chart — размер пузыря = количество мыслей, подпись = тема
- Drill-down: клик на пузырь → список мыслей этого кластера
- Trend: как менялся размер кластера по месяцам

**Сложность: XL**
Кластеризация + LLM-именование + bubble-chart визуализация + temporal analysis. Много движущихся частей.
