# PRD: Stream & Distillation — Автоматический захват разговоров

**Проект:** Open Brain v0.2  
**Автор:** Max Postnikov + Claude  
**Дата:** 5 марта 2026  
**Статус:** Draft

---

## 1. Проблема

Open Brain захватывает мысли только по явному запросу — через CLI, MCP tool `brain_save` или триггеры в Telegram. Разговоры с AI (Claude Desktop, claude.ai) — один из самых продуктивных источников мышления — не оставляют следа в системе, если не попросить об этом вручную.

Это создаёт парадокс: самые глубокие обсуждения — те, где рождаются инсайты, принимаются решения, формулируются вопросы — исчезают после закрытия чата. Ручное сохранение ломает поток мышления и требует переключения внимания с содержания на процесс захвата.

## 2. Решение

Двухслойная архитектура памяти, вдохновлённая механизмом консолидации памяти в человеческом мозге:

**Stream (поток)** — сырой захват всего, что происходит в разговоре. Лёгкий, дешёвый, без AI-обработки. Пишется тихо и непрерывно. Транзитная зона — не хранилище.

**Distillation (дистилляция)** — фоновый процесс, который периодически «засыпает», перечитывает накопленный поток и извлекает из него полноценные мысли с embedding и метаданными. Аналог ночной консолидации памяти: гиппокамп (поток) → неокортекс (thoughts).

### Метафора

Днём мозг записывает всё подряд. Ночью — перерабатывает, извлекает паттерны, укладывает важное в долговременную память, остальное забывает. Open Brain работает так же: Stream пишет непрерывно, Distillation спит и перерабатывает, Thoughts хранят результат.

## 3. Архитектура


### Взаимодействие с существующей архитектурой

Stream — новый слой, который не затрагивает текущую работу Thoughts. Все существующие MCP tools, CLI, Web UI, REST API продолжают работать как раньше. Явное `brain_save` по-прежнему пишет напрямую в Thoughts через полный pipeline (embedding + metadata).

Когда пользователь явно просит сохранить мысль в разговоре, она попадает и в Stream (как часть потока), и сразу в Thoughts (через стандартный capture pipeline).

## 4. Stream — подробный дизайн

### 4.1. Что записывается

Смысловые блоки разговора. Один блок — это один «раунд» обсуждения: постановка вопроса → обсуждение → промежуточный результат. Не отдельные сообщения, а связные фрагменты диалога.

### 4.2. Как определяются границы блока

LLM-клиент (Claude) определяет границу блока в момент генерации ответа по следующим сигналам:

- Тема сменилась
- Принято решение или сформулирован вывод
- Пользователь явно перешёл к новому вопросу
- Ответ длинный и самодостаточный — предыдущий обмен можно закрыть

Это judgement call, не детерминированный алгоритм. Для сырого потока, который будет дистиллирован, идеальная нарезка не нужна — нужна достаточно хорошая.

### 4.3. Формат блока

```
session: claude-2026-03-05-open-brain-architecture
block: 3
topic: гранулярность потока и механика сна
---
[Max]: описывает метафору сна и двухслойную архитектуру — stream 
как гиппокамп, distillation как ночная консолидация...
[Claude]: соглашается, предлагает stream без embedding, thoughts 
через полный pipeline. Вопрос: кто и когда спит?
[Max]: cron раз в сутки + power nap в Web UI...
```

Поля:
- `session` — идентификатор сессии разговора (генерируется при начале диалога)
- `block` — порядковый номер блока в сессии
- `topic` — краткое описание темы блока (генерируется Claude)
- `participants` — кто участвовал (пока Max + Claude, но расширяемо)
- Тело — свободный текст с маркерами участников

### 4.4. Свойства потока

- **Без embedding** — экономия на OpenAI, быстрая запись
- **Без metadata extraction** — без вызова gpt-4o-mini
- **Текстовый поиск** — PostgreSQL full-text search (`tsvector/tsquery`) или простой `ILIKE`
- **TTL 30 дней** — записи старше 30 дней удаляются автоматически (cron или при обращении)
- **Привязка к сессии** — блоки одного разговора связаны через `session_id`

### 4.5. Источники потока

На первом этапе — только разговоры с Claude через MCP. В будущем — любой клиент, который поддерживает MCP-протокол и может вызывать `stream_write`.

## 5. Distillation — подробный дизайн

### 5.1. Что делает дистилляция

Читает недистиллированные блоки из Stream, группирует по сессиям, передаёт LLM (gpt-4o-mini или gpt-4o) с промптом: «Извлеки значимые мысли из этого разговора». Результат — одна или несколько записей в Thoughts через стандартный capture pipeline (embedding + metadata).

### 5.2. Промпт дистилляции

LLM получает:

- Блоки одной сессии (или нескольких за день)
- Инструкцию: извлечь решения, инсайты, вопросы, формулировки, противоречия
- Формат вывода: массив мыслей, каждая с предложенным `content_type`

Дистилляция не обязана извлечь что-то из каждого блока. Если разговор был пустым — 0 мыслей. Если насыщенным — 5-10.

### 5.3. Типы извлекаемого

| Тип | Описание | Пример |
|-----|----------|--------|
| `decision` | Принятое решение с обоснованием | «Решил использовать двухслойную архитектуру stream + thoughts» |
| `insight` | Новое понимание | «Фильтрация должна быть на стороне хранилища, а не захвата» |
| `question` | Открытый вопрос для возврата | «Как масштабировать дистилляцию при 100+ сессиях в день?» |
| `formulation` | Точная формулировка или метафора | «Поток — это гиппокамп, дистилляция — ночной сон неокортекса» |
| `contradiction` | Изменение позиции | «Раньше думал, что нужен embedding на потоке — теперь считаю иначе» |

### 5.4. Когда запускается

**Автоматически (cron):** раз в сутки, ночью. Собирает все недистиллированные блоки за день в один батч, независимо от сессии — это даёт LLM возможность видеть связи между разными разговорами. Конфигурируемое время в `~/.open-brain/config.json`.

**Вручную (Power Nap):** кнопка в Web UI. Запускает дистилляцию прямо сейчас. Полезно после важного разговора, когда не хочешь ждать ночного cron.

**CLI:** `brain distill` — запуск из терминала.

### 5.5. Связь дистиллированных мыслей с потоком

Каждая мысль, созданная дистилляцией, содержит:
- `source: 'distillation'` — чтобы отличать от ручных сохранений
- `source_ref` — ссылка на session_id и block_id, из которых извлечена
- Стандартные теги + автоматический тег `distilled`

Это позволяет от мысли перейти к исходному разговору (пока поток не истёк).

### 5.6. Маркировка обработанного потока

После дистилляции блоки помечаются флагом `distilled_at` (timestamp). Это не удаляет их — они продолжают жить до истечения TTL. Но повторная дистилляция их пропускает.

## 6. Прозрачность и наблюдаемость

Принцип: пользователь в любой момент должен понимать, что происходит внутри Open Brain. Никаких скрытых процессов, никакой магии.

### 6.1. Web UI: вкладка «Stream»

Новая вкладка в Web UI, показывающая сырой поток:

- **Лента блоков** — хронологическая, сгруппированная по сессиям
- **Фильтры:** по сессии, по дате, по статусу (raw / distilled)
- **Статус каждого блока:** 
  - `raw` — ещё не дистиллирован (индикатор: серый)
  - `distilled` — обработан, есть ссылка на извлечённые мысли (индикатор: зелёный)
  - `expired` — до удаления осталось < 3 дней (индикатор: жёлтый, fade-эффект)
- **Связи:** от дистиллированного блока — ссылки на созданные мысли; от мысли — обратная ссылка на исходные блоки
- **Inline-просмотр:** раскрытие полного текста блока прямо в ленте
- **Ручные действия:** пометить блок как «важный» (не удалять по TTL), вручную отправить на дистилляцию

### 6.2. Web UI: вкладка «Distillation Log»

Полный лог каждого запуска дистилляции:

- **Когда запущен:** timestamp, триггер (cron / power nap / CLI)
- **Что обработал:** количество блоков, количество сессий
- **Что извлёк:** список созданных мыслей со ссылками
- **Что пропустил:** блоки, из которых не было извлечено ничего (с объяснением LLM, почему)
- **Стоимость:** количество токенов, примерная стоимость в $
- **Длительность:** время выполнения
- **Ошибки:** если были — полный текст

Лог хранится в таблице `distillation_log`.

### 6.3. Web UI: Dashboard / Status

Обновлённая вкладка Stats (или отдельная вкладка «Brain Status»), которая показывает текущее состояние системы в реальном времени:

**Поток:**
- Блоков в потоке (всего / raw / distilled)
- Блоков, истекающих в ближайшие 3 дня
- Последняя запись в поток: когда, из какой сессии
- Средний объём потока в день

**Дистилляция:**
- Последний запуск: когда, триггер, результат
- Следующий запланированный запуск (cron)
- Извлечено мыслей за последнюю неделю
- Соотношение: блоков в потоке → мыслей извлечено (конверсия)
- Накопленная стоимость дистилляции за 7/30 дней

**Общее здоровье:**
- Thoughts: всего, за 7 дней, за 30 дней (как сейчас)
- Stream: объём, скорость роста
- Storage: размер базы, прогноз роста
- OpenAI: расход за текущий месяц по операциям (embed / metadata / distill)

### 6.5. Уведомления в Web UI

Ненавязчивые уведомления:
- После каждой дистилляции: «Извлечено N мыслей из M блоков» (toast notification)
- Когда поток пуст > 3 дней: «Поток пуст. Последний разговор — N дней назад»
- Когда приближается истечение непереработанных блоков: «5 блоков истекут завтра, не дистиллированы»

## 7. База данных

### 7.1. Новая таблица: `stream`

| Столбец | Тип | Описание |
|---------|-----|----------|
| `id` | UUID | Уникальный идентификатор (auto) |
| `session_id` | VARCHAR(255) | ID сессии разговора |

| `topic` | TEXT | Краткое описание темы блока |
| `content` | TEXT | Полный текст блока с маркерами участников |
| `participants` | TEXT[] | Массив участников (e.g., ['max', 'claude']) |

| `pinned` | BOOLEAN | Защита от TTL-удаления (default: false) |

| Индекс | Тип | Столбец |
|--------|-----|---------|
| `idx_stream_session` | B-tree | session_id |

| `idx_stream_distilled` | B-tree (partial) | distilled_at (WHERE distilled_at IS NULL) |
| `idx_stream_fulltext` | GIN | to_tsvector('simple', content) |

Новые значения для существующих полей:

Новых столбцов не требуется.

## 8. MCP Tools

### 8.1. Новый tool: `stream_write`

Запись блока в поток. Вызывается LLM-клиентом (Claude) молча, по ходу разговора.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `session_id` | string | да | ID текущей сессии |
| `block_number` | number | да | Порядковый номер блока |
| `topic` | string | да | Краткое описание темы (5-15 слов) |
| `content` | string | да | Текст блока с маркерами участников |
| `participants` | string[] | нет (['user', 'claude']) | Участники |

Этот tool **не вызывает OpenAI API** — только INSERT в PostgreSQL. Быстрый и дешёвый.

### 8.2. Новый tool: `stream_read`

Чтение потока. Для случаев, когда Claude хочет вспомнить контекст предыдущих сессий.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `session_id` | string | нет | Фильтр по сессии |
| `limit` | number | нет (20) | Количество блоков |
| `status` | string | нет | raw / distilled / all |

### 8.3. Обновление brain_save

Когда `brain_save` вызывается во время активной сессии (есть `session_id` в контексте), мысль дополнительно получает ссылку на текущую сессию в `source_ref`. Это позволяет связать явно сохранённые мысли с контекстом разговора.

## 9. CLI

### 9.1. Новые команды

```bash
# Ручной запуск дистилляции
brain distill

# Запись в поток (для интеграции с OpenClaw и другими CLI-клиентами)
brain stream-write --session <id> --topic "тема блока" "текст блока"

# Просмотр потока
brain stream                    # последние 20 блоков
brain stream --session <id>     # блоки конкретной сессии
brain stream --status raw       # только необработанные

# Статус системы
brain status                    # сводка: поток, дистилляция, thoughts
```

## 10. Конфигурация

Новая секция в `~/.open-brain/config.json`:

```json
{
  "stream": {
    "ttl_days": 30,
    "cleanup_on_startup": true
  },
  "distillation": {
    "enabled": true,
    "schedule": "0 3 * * *",
    "model": "gpt-4o-mini",
    "temperature": 0.3,
    "max_blocks_per_run": 200,
    "min_block_length": 50
  }
}
```

| Параметр | Описание | По умолчанию |
|----------|----------|-------------|
| `stream.ttl_days` | Время жизни блоков в потоке | 30 |
| `stream.cleanup_on_startup` | Удалять просроченные блоки при старте сервера | true |
| `distillation.enabled` | Включена ли автоматическая дистилляция | true |
| `distillation.schedule` | Cron-выражение для расписания | `0 3 * * *` (3:00 ночи) |
| `distillation.model` | Модель для дистилляции | gpt-4o-mini |
| `distillation.temperature` | Temperature для LLM | 0.3 |
| `distillation.max_blocks_per_run` | Максимум блоков за один запуск | 200 |
| `distillation.min_block_length` | Минимальная длина блока для обработки (символов) | 50 |

## 11. Стоимость

### Ежедневное использование (5 разговоров, ~30 блоков)

| Операция | Токены/день | Стоимость/день |
|----------|------------|---------------|
| Stream запись | 0 | $0 (только PostgreSQL) |
| Дистилляция (input: ~30 блоков) | ~30K | ~$0.005 |
| Дистилляция (output: ~5 мыслей) | ~2K | ~$0.001 |
| Embedding для извлечённых мыслей | ~5K | ~$0.0001 |
| Metadata для извлечённых мыслей | ~5K | ~$0.001 |
| **Итого дополнительно** | | **~$0.007/день** |

Текущие расходы (~$0.002/день) + дистилляция (~$0.007/день) = **~$0.01/день** или **~$0.30/мес**.

## 14. Решённые вопросы

1. **Мультиязычный full-text search:** конфигурация `simple` без стемминга. Работает одинаково для русского и английского, не требует определения языка.

2. **Размер контекста дистилляции:** батчами. Все недистиллированные блоки за день собираются в один контекст для LLM, независимо от сессии. Это даёт дистиллятору возможность видеть связи между разными разговорами за день.

3. **Качество дистилляции:** формального feedback loop нет. Если качество не устроит — ручная доработка промпта по логам дистилляции в разговоре с Claude. Промпт живёт в коде (`src/distillation/prompt.ts`), итеративно улучшается.

4. **Несколько параллельных сессий:** разные session_id, конфликтов нет. Edge cases проверяются при реализации.

5. **Дистилляция Telegram-потока:** Phase 4. OpenClaw пишет в Stream через CLI (`brain stream-write`), дистилляция обрабатывает наравне с остальными блоками.
